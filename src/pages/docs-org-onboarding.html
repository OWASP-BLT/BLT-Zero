Organization Onboarding

What Organization Admin Does

  1. 1. Generate keypair locally using the provided script (private key stays with your organization)
  2. 2. Publish the public key at a well-known URL on your domain — no contact with BLT-Zero required
  3. 3. Decrypt incoming vulnerability reports locally with your private key

Important: Your private key never leaves your infrastructure. Only you can decrypt the vulnerability reports.

Self-Service Key Publishing (Recommended)

BLT-Zero automatically discovers your public key from a predictable URL on your domain. No registration or contact with BLT-Zero is required.

Well-Known URL

If your security contact is security@example.com, host your public key at: 

https://example.com/.well-known/blt-zero/public_key.jwk

The file must be served over HTTPS with Content-Type: application/json.

🔑 When your key is published: The browser ECDH-wraps the ZIP password with your public key before sending anything to the server. The BLT-Zero Worker never sees the plaintext password. Only your private key can recover it.

Available Scripts

We provide Python scripts in the BLT-Zero repository under tools/ directory.

1. Generate Keypair

python tools/org_keygen.py

Generates private_key.jwk (keep secret) and public_key.jwk (publish at well-known URL).

2. Recover ZIP Password (Phase 2)

Copy the JSON block from the received email and save it as blt-zero-key.json, then run: 

python tools/org_decrypt.py private_key.jwk blt-zero-key.json

Prints the recovered ZIP password. Use it to open the attached .zip file.
